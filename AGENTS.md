# scel2rime Agent Guidance

## Project intent
- `scel2rime` converts Sogou `.scel` cell dictionaries into RIME `.dict.yaml` files for macOS RIME users.
- Primary sample dictionary: Sogou 网络流行新词 (`id=4`).
- RIME reference style: `gaboolic/rime-frost` dictionaries: UTF-8 YAML header, then tab-separated `word<TAB>pinyin<TAB>weight` rows.

## Rust workflow
- Use repo-pinned nightly Rust from `rust-toolchain.toml`.
- Before finishing changes, run:
  - `cargo fmt -- --check`
  - `cargo clippy --all-targets -- -D warnings`
  - `cargo test`
  - `cargo check`
- Install missing nightly components with `rustup component add rustfmt clippy` when needed.
- Prefer small, readable Rust over clever Rust. This repo is also a Rust learning project.

## Code structure preference
- Keep CLI thin in `src/main.rs`.
- Put parsing, data model, and rendering in `src/lib.rs` or modules under `src/`.
- Split later when size warrants it:
  - `src/scel.rs` for binary SCEL parsing.
  - `src/rime.rs` for RIME YAML rendering.
  - `src/cli.rs` for argument parsing.
- Unit-test parser helpers with generated byte buffers where possible.
- Use ignored real `.scel` files only for manual/local verification, not committed fixtures.

## Crate preference
- For Rust projects generally, prefer `snafu` for both library and application error handling.
- For this project specifically, stay std-only unless a dependency pays for itself; `scel2rime` is intentionally minimal.
- Add crates when they remove meaningful boilerplate:
  - `snafu` for richer errors in larger Rust projects.
  - `clap` for CLI once flags/options appear.
  - `tempfile` as dev-dependency for integration tests needing files.
- Avoid broad framework crates for this small converter.

## Error handling
- Library code returns `Result<T, Error>`; no `unwrap`, `expect`, `panic`, or `assert` for user-controlled files.
- In this project, implement `std::error::Error` manually to avoid third-party dependencies.
- In non-minimal Rust projects, use `snafu` instead of `thiserror`/`anyhow` as default preference.
- Binary prints concise errors and exits non-zero.
- Validate offsets and lengths before slicing binary data.
- Error messages should name corrupt section/offset when practical.

## SCEL parsing notes
- Sogou files can use old `DCS` header bytes or newer `ECS` bytes.
- Pinyin table starts at `0x1544` after a 4-byte count/header at `0x1540`.
- Word table starts dynamically after pinyin table, not always at `0x2628`.
- Word records contain:
  - homophone count (`u16`)
  - pinyin index byte length (`u16`)
  - pinyin indices (`u16` each)
  - repeated words: word byte length (`u16`), UTF-16LE word, extension byte length (`u16`), frequency (`u16`), extension tail.

## Config preference
- Batch config is ID-focused and std-only: one Sogou dictionary per line as `<id> <name>`.
- Keep `scel2rime.conf` human-editable; comments start with `#`.
- Avoid adding HTTP/config crates for now; config mode uses `curl` to download SCEL bytes into memory.

## Output preference
- Write `luna_pinyin.sogou.<stem>.dict.yaml` for local files.
- Write `dist/luna_pinyin.sogou.<id>.dict.yaml` for config-downloaded dictionaries.
- Header should be valid RIME dictionary YAML:
  - comments
  - `---`
  - `name:`
  - `version:`
  - `sort: by_weight`
  - `use_preset_vocabulary: true`
  - `...`
- Entries use tab-separated `word`, pinyin syllables separated by spaces, then numeric weight.
